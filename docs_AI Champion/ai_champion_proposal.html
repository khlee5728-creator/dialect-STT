2. 기술 명세

2.1 기술목적

국내 독거노인 약 190만 명 중 상당수가 사투리를 주로 사용하지만, 기존 AI 음성 서비스는 표준어 중심으로 설계되어 방언 화자의 인식률이 크게 떨어진다. 본 기술은 중·노년층 방언 음성을 정확히 인식하고 자연스러운 방언으로 응답하여, 어르신의 고독감과 우울감을 경감하는 AI 말동무 서비스를 구현한다. 더불어 대화 히스토리를 누적·분석하여 어르신의 인생사와 관심사를 장기 기억하고, 가족에게 일일 대화 요약과 디지털 회고록을 제공하는 3자 연결 케어 플랫폼을 목표로 한다.

2.3 주요기능

  • 방언 음성인식: 경상/전라/충청/제주 4대 방언 전용 STT (AI Hub 중·노년층 데이터 활용)
  • 방언 어미 TTS 응답: 경상(~했심더, ~이라예, ~했능교) / 전라(~했제, ~이여, ~했당께) / 충청(~했슈, ~이유, ~했쥬)
  • AI 페르소나 대화: "친근한 손녀·손자" 캐릭터(예: "하늘이"), 사투리 권역별 맞춤 따뜻한 말투, 노인 발화 속도 맞춤(침묵 3초 이상 허용, 한 번에 질문 하나)
  • 5단계 대화 구조 (하루 1회 10~15분): ① 오프닝(인사·날씨) → ② 감정 체크 → ③ 메인 대화(요일별 테마) → ④ 건강·복약 체크 → ⑤ 클로징(가족 근황)
  • 요일별 콘텐츠 테마: 월(일상안부)·화(인생회상)·수(인지훈련)·목(가족근황)·금(건강관리)·토(회고록)·일(자유대화)
  • 장기 기억 3중 구조: 기본 프로필 + 최근 7일 대화 요약 + 회고록 누적 → LLM 프롬프트 주입으로 "지난번에 말씀하셨는데…" 자연스러운 연속성
  • 4종 알림 시스템: ① 미수신 감지 ② 감정 이상 감지(positive/neutral/negative/alert) ③ 일일 리포트 ④ 장기 미응답(고독사 예방)
  • 가족 3자 연결: 카카오 알림톡(1순위) + SMS fallback으로 가족·복지사에게 즉시 전달
  • 디지털 회고록: 인생 주제별 대화(고향→어린시절→직업→자녀양육→인생의 지혜) 텍스트 누적 → 주간 가족 리포트
  • 양방향 전화 인터페이스: AI가 매일 먼저 전화(Outbound) + 어르신이 원할 때 전화(Inbound), 앱 설치 불필요

2.4 결과물 형상

  • 어르신용: Twilio 양방향 전화 인터페이스 (Outbound 자동 발신 + Inbound 수신)
  • 가족용: React Native 모바일 앱 (iOS/Android) — 일일 리포트, 미수신 알림
  • 관리자용: 웹 대시보드 (대화 통계, 감정 분석 모니터링)

2.5 배포방식

  • 전화 인터페이스: Twilio API 기반 양방향 전화 (별도 설치 불필요)
  • 모바일 앱: App Store / Google Play 배포
  • B2G/B2B: API 제공 + 온프레미스 설치 지원

2.6 혁신적 요소

  • 방언 특화 STT 파인튜닝: AI Hub 중·노년층 한국어 방언 데이터로 Whisper 모델을 파인튜닝하여 기존 범용 STT 대비 방언 인식률을 획기적으로 개선
  • 사투리 어미 치환 TTS: LLM 응답을 지역별 방언 어미로 자동 변환하는 후처리 레이어로, TTS 재학습 없이 방언 응답 구현
  • 노인 친화 페르소나 + 대화 품질 원칙: "친근한 손녀·손자" 캐릭터, 침묵 허용·반복 수용·한 번에 질문 하나·강요 금지 등 기존 AI 챗봇과 차별화된 노인 발화 특성 대응 대화 설계
  • 요일별 콘텐츠 테마 + 인지 훈련 통합: 단순 안부 대화를 넘어 인생 회상·인지 훈련(치매 예방)·회고록 기록을 자연스럽게 통합한 케어 콘텐츠 설계
  • 장기 기억 3중 아키텍처: 기본 프로필 + 최근 7일 대화 요약 + 회고록 누적 텍스트를 매 대화마다 LLM 프롬프트에 주입하여 수개월~수년 맥락 유지
  • 다채널 알림 통합: 카카오 알림톡(1순위, 건당 8~15원) + SMS(Twilio fallback) 자동 전환으로 가족·복지사에게 안정적 전달
  • LLM 기반 감정 분석: 부정적/위기 발화 패턴을 4단계(positive/neutral/negative/alert)로 분류하여 alert 감지 시 즉시 가족 알림

2.7 도전적 요소

  • 실사용 데이터 기반 모델 지속 개선: 서비스 운영을 통해 축적되는 실사용 대화 데이터를 핵심 자산으로 활용, 시간이 갈수록 기술 경쟁력이 강화되는 자기강화형 시스템 구축
  • 노인 음성 특화: 고령자의 느린 발화 속도, 약한 자음 발성, 음질 저하 등 일반 STT가 처리하기 어려운 음향적 특징에 대응하는 모델 최적화
  • 노인 친화 대화 설계의 정량화: "침묵 3초 허용", "한 번에 질문 하나", "반복 수용", "강요 금지" 등 기존 AI 챗봇이 위반하는 노인 발화 특성을 LLM 프롬프트·대화 정책으로 정량화·자동화
  • 인지 훈련의 자연스러운 통합: 테스트 느낌 없이 게임처럼 자연스러운 회상·언어·지남력·주의력 훈련을 일상 대화에 녹여내는 콘텐츠 엔지니어링
  • 소멸 위기 방언 보존: 제주어 등 소멸 위기 방언까지 포함한 디지털 아카이빙과 실시간 인식 기술은 학술적으로도 미개척 영역
  • 치매 조기 감지: 장기 대화 패턴 + 인지 훈련 반응 패턴 DB 분석을 통한 언어능력 변화 감지 → 인지장애 초기 징후 포착 가능성 (의료 AI 연계)

3. 구현 방법 및 계획

3.1 구현범위

세부업무 #1 방언 STT 엔진 개발 — AI Hub 데이터 전처리, Whisper 파인튜닝 (경상도 우선), 사투리→표준어 후처리 사전 구축, Deepgram 8kHz 전화 음질 연동, CER 기준 벤치마크 평가
세부업무 #2 양방향 전화 대화 + 알림 시스템 구축 — Twilio Outbound/Inbound API 연동, APScheduler 발신 스케줄러, Claude Haiku 사투리 프롬프트 설계, 장기 기억 저장 구조(대화 요약→DB), Clova Voice TTS + 사투리 어미 치환 레이어, 4종 알림 시스템(미수신/감정이상/일일리포트/장기미응답) 및 LLM 기반 감정 분석(positive/neutral/negative/alert), 카카오 알림톡+SMS fallback 발송 로직
세부업무 #3 가족 앱 및 통합 테스트 — React Native 보호자용 앱, 일일 대화 요약 리포트, 디지털 회고록 자동 생성, 감정 이상 감지 알림, 전 구간 통합 테스트 (지연시간 1.3초 이내 검증)

4. 파급효과

4.1 기술적 파급효과

  • 한국어 방언 STT 벤치마크 수립: 경상/전라/충청/제주 4대 방언에 대한 체계적 음성인식 성능 평가 기준 제시, 관련 논문 발표 가능
  • 노인 음성 특화 AI 모델: 고령자 발화 특징(느린 속도, 약한 자음, 음질 저하)에 대응하는 음성인식 최적화 기술로 특허 출원 가능
  • 소멸 위기 방언 디지털 아카이빙: 제주어 등 소멸 위기 방언의 음성 데이터를 체계적으로 수집·보존하는 학술적 기여
  • 기술 라이선싱·OEM 공급 가능: 사투리 음성 AI 기술을 외부 API로 공급하여 디바이스·플랫폼 파트너(효돌㈜, KT/SK/LG 셋톱박스 등)에게 라이선싱하는 기술 기반 서비스 회사로의 확장

4.2 사회·산업적 파급효과
