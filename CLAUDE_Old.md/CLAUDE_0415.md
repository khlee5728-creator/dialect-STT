# 프로젝트: 사투리 AI 말동무 서비스 (어르신 대상)

## 프로젝트 전략 방향

### 핵심 포지셔닝
> **국내 유일 사투리 특화 AI 말동무 서비스**
>
> 기존 노인 돌봄 AI가 해결하지 못한 두 가지 문제를 동시에 해결한다.
> - **언어 장벽**: 표준어 중심 AI는 사투리를 쓰는 어르신과 제대로 대화하지 못함
> - **관계 단절**: 주 1회 3분짜리 안부 확인은 진짜 말동무가 될 수 없음
>
> 사투리 STT/TTS 기술을 자체 개발·고도화하며,
> 서비스 운영을 통해 축적되는 실사용 데이터가 기술 경쟁력을 지속적으로 강화한다.
> 장기적으로는 사투리 음성 AI 기술을 디바이스·플랫폼 파트너에게 공급하는
> **기술 기반 서비스 회사**로 성장한다.

### 전략 로드맵
```
1단계 — 서비스 출시 및 시장 검증
  사투리 STT 자체 개발 (경상도 우선)로 기술 차별화 확보
  전화 기반 AI 말동무 서비스 출시
  어르신·가족·지자체 레퍼런스 확보
  실사용 대화 데이터 축적 → 기술 고도화 재료

2단계 (6개월~1년) — 기술·서비스 동시 고도화
  실사용 데이터 기반 사투리 STT 전 권역 확장
  가족 앱 출시, 디바이스 파트너십 협의 시작

3단계 (장기) — 기술 사업화 확장
  사투리 음성 AI API 외부 공급 (효돌㈜ 등 디바이스 업체)
  지자체 화이트라벨 납품
  특허 기반 기술 라이선싱
```

## 서비스 개요

실버산업 AI 적용 프로젝트. 어르신 대상 AI 말동무 서비스에 **사투리 STT/TTS**를 접목해
기존 서비스와 차별화된 노인 케어 플랫폼을 개발한다.

### 기존 서비스와의 차별화
| 기존 서비스 (효돌, KT AI케어 등) | 우리 서비스 |
|---|---|
| 주 1회 안부 전화, 3분 수준 | 매일 충분한 대화 |
| 표준어 중심 | 사투리 이해·응답 |
| 대화 내용 휘발 | 장기 기억 누적 |
| 어르신만 대상 | 어르신 + 가족 3자 연결 |
| 정해진 스크립트 | 맥락 있는 자연스러운 대화 |

### 핵심 차별화 포인트
- **사투리 이해**: 경상/전라/충청/제주 방언 인식 및 응답
- **장기 기억**: 대화 히스토리 누적 → 어르신 인생/관심사 기억
- **가족 연결 3자 구조**: 어르신 ↔ AI ↔ 가족 앱 연동
- **디지털 회고록**: 대화 기반 자동 인생 기록 생성
- **양방향 전화 인터페이스**: 설치·앱 불필요, 일반 전화기만으로 사용
  - **메인 (Outbound)**: AI가 매일 정해진 시간에 어르신에게 먼저 발신
  - **서브 (Inbound)**: 어르신이 생각날 때 언제든 전화 가능

## 인터페이스 전략

### MVP — 전화 (즉시 출시 가능)
- 하드웨어 불필요, 기술 검증에 집중
- 어르신이 이미 익숙한 전화기 활용
- B2G 납품 시 디바이스 없이 바로 계약 가능
- **단점 인지**: 전화가 rapport 형성에 한계 있음 → 2단계에서 보완

### 2단계 — 디바이스 확장 (파트너십)
- 직접 제조 없이 기존 업체와 협력
  - 효돌㈜에 사투리 STT/TTS 기술 탑재 협의
  - TV 셋톱박스 앱 (KT/SK/LG 협력)
  - 스마트 액자·AI 스피커 연동
- 핵심: **우리가 디바이스를 만드는 게 아니라 기술을 얹는 구조**

### 인터페이스별 특성
| 인터페이스 | Rapport | 접근성 | 비용 | 시점 |
|---|---|---|---|---|
| 전화 | ⭐⭐ | ⭐⭐⭐⭐⭐ | 최저 | MVP |
| TV 앱 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 낮음 | 2단계 |
| 인형 디바이스 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ | 높음 | 파트너십 |
| 카카오톡 | ⭐⭐ | ⭐⭐⭐⭐⭐ | 최저 | 보조채널 |

## 기술 스택

### MVP 기준
- **STT**: Whisper + AI Hub 방언 데이터 기본 파인튜닝 (1개 권역 우선) + 사투리 후처리 사전 병행
- **LLM**: Claude API (claude-haiku — 빠른 응답)
- **TTS**: Clova Voice + 사투리 어미 치환 (프롬프트 레벨)
- **전화**: Twilio 양방향 API
- **서버**: FastAPI + WebSocket
- **DB**: PostgreSQL (대화 요약·감정 상태 저장)
- **가족 앱**: React Native

### 사투리 처리 전략 (단계별)
```
MVP — 기본 파인튜닝 + 후처리 병행 (차별화 확보)
  AI Hub 경상도 방언 데이터 → Whisper 기본 파인튜닝 (1개 권역)
  + 사투리→표준어 후처리 사전으로 인식률 보완
  → LLM → 사투리 어미 TTS
  ※ 완벽하지 않아도 됨 — 타 서비스 대비 사투리 인식 자체가 차별점

2단계 — 파인튜닝 확장 (실사용 데이터 추가)
  AI Hub 전 권역 데이터 + 실사용 데이터 → Whisper 파인튜닝 고도화
  전라도 → 충청도 → 제주도 순차 확장
  인식률 벤치마크 (CER 기준) 지속 측정

3단계 — 풀스택 고도화 (서비스 안정화 후)
  노인 음성 특화 STT (느린 발화·약한 발음 대응)
  사투리 TTS 파인튜닝
  특허 출원 → 외부 API 공급 검토
```

### TTS 사투리 어미 (MVP 즉시 적용)
- 경상도: ~했심더, ~이라예, ~했능교
- 전라도: ~했제, ~이여, ~했당께
- 충청도: ~했슈, ~이유, ~했쥬

## Voice AI 파이프라인

### 전화 방식
```
[메인] Outbound — AI가 먼저 전화
스케줄러 (매일 오전 10시)
      ↓
Twilio Outbound Call → 어르신 발신
      ↓
어르신 수신 → 대화 시작

[서브] Inbound — 어르신이 전화
어르신이 서비스 번호로 전화
      ↓
Twilio Inbound 수신 → 대화 시작
```

### 공통 대화 흐름
```
대화 시작
      ↓
STT (Deepgram, 전화 음질 8kHz 최적화)
      ↓
사투리 후처리 정규화
      ↓
LLM (Claude Haiku + 장기 기억 컨텍스트)
      ↓
TTS (Clova Voice + 사투리 어미 변환)
      ↓
어르신에게 음성 응답
      ↓
대화 종료 → DB 저장 (요약 + 감정 상태)
      ↓
가족 앱 일일 리포트 발송
```

### 주요 시나리오
- **정기 안부**: 매일 오전 10시 AI 먼저 발신
- **복약 알림**: "어르신, 약 드실 시간이에요"
- **기념일 케어**: "오늘 손녀 생일이라고 하셨잖아요~"
- **미수신 감지**: 2회 이상 무응답 → 가족/복지사 자동 알림
- **감정 이상 감지**: 부정적 발화 패턴 → 가족에게 알림

### Latency 목표
```
STT:  ~300ms  (Deepgram 스트리밍)
LLM:  ~800ms  (Claude Haiku)
TTS:  ~200ms  (스트리밍 TTS)
합계: ~1.3초  (허용 범위 2초 이내)
```

## 알림 시스템 (가족·복지사)

### 알림 트리거
```
1. 미수신 감지    — 2회 이상 전화 안 받음
2. 감정 이상 감지 — "외롭다", "살기 싫다" 등 부정적 발화 패턴
3. 일일 리포트    — 매일 대화 종료 후 요약 자동 전송
4. 장기 미응답    — 며칠째 대화 없음 (고독사 예방)
```

### 알림 채널 전략 (MVP)
| 채널 | 구현 난이도 | 친숙도 | MVP 적합성 |
|---|---|---|---|
| **카카오 알림톡** | 쉬움 | ⭐⭐⭐⭐⭐ | ✅ 1순위 |
| **SMS** | 매우 쉬움 | ⭐⭐⭐⭐⭐ | ✅ fallback |
| 앱 푸시 (React Native) | 중간 | ⭐⭐⭐ | 2단계 |
| 이메일 | 매우 쉬움 | ⭐⭐ | 보조 |

- **카카오 알림톡**: 메인 채널 — 가족·복지사 모두 카카오톡 사용, 건당 약 8~15원
- **SMS (Twilio)**: 카카오톡 미설치 또는 발송 실패 시 자동 fallback

### 알림 메시지 유형

**① 미수신 알림**
```
[AI 말동무] 홍길동 어르신께 오전 10시, 10시 15분
두 차례 전화를 드렸으나 받지 않으셨어요.
직접 확인해 보시겠어요?
```

**② 감정 이상 감지**
```
[AI 말동무] 오늘 홍길동 어르신과의 대화에서
평소와 다른 표현이 감지되었어요.
대화 요약 보기 → [링크]
```

**③ 일일 리포트**
```
[AI 말동무] 오늘 홍길동 어르신 대화 요약
통화시간: 12분 | 기분: 좋음
"손녀 결혼식 얘기를 많이 하셨어요"
전체 내용 보기 → [링크]
```

**④ 장기 미응답 알림**
```
[AI 말동무] 홍길동 어르신과 3일째
대화가 이루어지지 않고 있어요.
안부 확인이 필요할 수 있습니다.
```

### 알림 전체 흐름
```
대화 종료 or 미수신 발생
      ↓
FastAPI 서버에서 트리거 감지
      ↓
LLM으로 대화 요약 + 감정 분석
      ↓
DB 저장 (요약·감정 상태·알림 이력)
      ↓
카카오 알림톡 발송
(실패 시 SMS fallback 자동 전환)
      ↓
가족/복지사 수신
      ↓
링크 클릭 → 웹 대화 요약 페이지
(MVP: 간단한 웹뷰 / 2단계: 가족 앱)
```

### 감정 분석 기준 (LLM 프롬프트)
- 부정적 감지어: "외롭다", "힘들다", "살기 싫다", "아프다", "무섭다"
- 위기 감지어: "죽고 싶다", "아무도 없다", "밥을 못 먹었다"
- 감정 상태 분류: positive / neutral / negative / alert
- alert 감지 시 즉시 가족·복지사에게 알림 발송

## 운영 비용 구조

### 통화 1분당 변동비
| 항목 | 서비스 | Outbound | Inbound |
|------|--------|----------|---------|
| 전화 | Twilio | ~18원 | ~12원 |
| STT | Deepgram | ~20원 | ~20원 |
| LLM | Claude Haiku | ~15원 | ~15원 |
| TTS | Clova Voice | ~10원 | ~10원 |
| **합계** | | **~63원/분** | **~57원/분** |

### 손익분기점 시뮬레이션 (어르신 100명, 1인 하루 10분)
```
변동비: 63원 × 10분 × 100명 × 30일 = 189,000원/월
고정비 (서버·DB·번호):            약  50,000원/월
총 원가:                          약 240,000원/월 → 1인당 2,400원
구독료 5,000~10,000원 기준 → 수익 확보 가능
```

## 주의사항 / 개발 원칙

- **서비스 우선**: 기술 완성을 기다리지 말고 현재 가능한 수준으로 빠르게 출시
- **데이터가 핵심 자산**: 실사용 대화 데이터 축적이 장기 경쟁력
- 어르신 사용자 기준: 디지털 문맹 가정, UX 극단적 단순화
- 사투리 응답 시 과장되지 않게 — 자연스러운 말투 수준 유지
- 개인정보: 음성 데이터 로컬 처리 우선, 클라우드 전송 최소화
- AI Hub 데이터 라이선스 확인 필수 (상업적 이용 조건)
- Twilio 한국 번호 등록 시간 여유 두고 진행 (수일 소요 가능)
