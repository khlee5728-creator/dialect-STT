# 프로젝트: 사투리 특화 AI 말동무 서비스 (어르신 대상)

## 프로젝트 개요

실버산업 AI 적용 프로젝트로, 어르신 대상 AI 말동무 서비스에 **사투리 STT/TTS**를 접목해
기존 서비스와 차별화된 노인 케어 플랫폼을 개발한다.

### 핵심 차별화 포인트
- **장기 기억**: 대화 히스토리 누적 → 어르신 인생/관심사 기억
- **사투리 이해**: 경상/전라/충청/제주 방언 인식 및 응답
- **가족 연결 3자 구조**: 어르신 ↔ AI ↔ 가족 앱 연동
- **디지털 회고록**: 대화 기반 자동 인생 기록 생성
- **양방향 전화 인터페이스**: 설치·앱 불필요, 일반 전화기만으로 사용 가능
  - **메인: AI가 먼저 전화 (Outbound)** — 매일 정해진 시간에 AI가 어르신에게 발신
  - **서브: 어르신이 전화 (Inbound)** — 어르신이 생각날 때 언제든 전화 가능

---

## 기술 스택

### STT (Speech-to-Text)
- **베이스 모델**: OpenAI Whisper (파인튜닝 가능 오픈소스)
- **데이터**: AI Hub 중·노년층 한국어 방언 데이터
  - 경상도 방언 (50대 이상 발화자)
  - 충청도·전라도·제주도 방언 (50대 이상 발화자)
- **전략**: 범용 STT + 사투리 후처리 정규화 레이어 (단기) → Whisper 파인튜닝 (중기)
- **전화 음질 대응**: Deepgram (8kHz 전화망 최적화, 실시간 스트리밍)

### LLM
- **모델**: Claude API (claude-haiku — 빠른 응답 우선)
- **프롬프트 전략**:
  - 사투리 표현 few-shot 예시 포함
  - 어르신 말투·관심사 컨텍스트 주입
  - 장기 기억 요약 포함

### TTS (Text-to-Speech)
- **단기**: 표준어 TTS + 사투리 어미 치환 (프롬프트 레벨)
  - 경상도: ~했심더, ~이라예
  - 전라도: ~했제, ~이여
  - 충청도: ~했슈, ~이유
- **중기**: 사투리 어미 특화 TTS 파인튜닝
- **장기**: 노인 음성 특징(느린 속도·낮은 음역) 반영 엔드투엔드 모델

### 인프라
- React Native (모바일 앱 — 가족용)
- **Twilio 양방향 전화 API**
  - Outbound (메인): AI → 어르신 발신, 매일 스케줄 발신 (APScheduler)
  - Inbound (서브): 어르신 → AI 수신, 어르신이 원할 때 언제든 전화 가능
  - 전화 미수신 시 가족 앱으로 자동 알림
  - 발신 전 카카오톡 선알림 연동 검토
- 서버: FastAPI + WebSocket
- DB: PostgreSQL (대화 요약·감정 상태 저장)

## Voice AI 파이프라인 (양방향 전화)

### 전화 방식
```
[메인] Outbound — AI가 먼저 전화
스케줄러 (매일 오전 10시)
      ↓
Twilio Outbound Call → 어르신 전화 발신
      ↓
어르신 수신 → 대화 시작

[서브] Inbound — 어르신이 전화
어르신이 서비스 번호로 전화
      ↓
Twilio Inbound 수신 → 대화 시작
```

### 공통 대화 흐름
```
대화 시작 (Outbound 또는 Inbound)
      ↓
STT (Deepgram, 전화 음질 8kHz 최적화)
      ↓
LLM (Claude Haiku — 빠른 응답)
      ↓
TTS (Clova Voice + 사투리 어미 변환)
      ↓
어르신에게 음성 응답
      ↓
대화 종료 → DB 저장 (요약 + 감정 상태)
      ↓
가족 앱으로 일일 리포트 발송
```

### 주요 시나리오
- **정기 안부**: 매일 오전 10시 발신
- **복약 알림**: "어르신, 약 드실 시간이에요"
- **기념일 케어**: "오늘 손녀 생일이라고 하셨잖아요~"
- **미수신 감지**: 2회 이상 무응답 → 가족/복지사 자동 알림
- **감정 이상 감지**: 부정적 발화 패턴 → 가족에게 알림

### 지연시간 (Latency) 목표
```
STT:  ~300ms  (Deepgram 스트리밍)
LLM:  ~800ms  (Claude Haiku)
TTS:  ~200ms  (스트리밍 TTS)
합계: ~1.3초  (어르신 허용 범위 2초 이내)
```

## 사투리 STT 구현 전략 (단계별)

```
1단계 — MVP (빠른 검증)
  범용 Whisper → 사투리 후처리 사전(dict) → LLM → 표준어 TTS
  어미만 사투리풍으로 조정

2단계 — 차별화
  AI Hub 방언 데이터로 Whisper 파인튜닝
  경상도 우선 → 전라도 → 충청도 순

3단계 — 풀스택
  노인 음성 특화 STT (느린 발화·약한 발음 대응)
  사투리 TTS 파인튜닝
  특허 출원
```

## 운영 비용 구조

### 통화 1분당 변동비
| 항목 | 서비스 | Outbound | Inbound |
|------|--------|----------|---------|
| 전화 | Twilio | ~18원 | ~12원 |
| STT | Deepgram | ~20원 | ~20원 |
| LLM | Claude Haiku | ~15원 | ~15원 |
| TTS | Clova Voice | ~10원 | ~10원 |
| **합계** | | **~63원/분** | **~57원/분** |

### 월 고정비
```
서버 (AWS/Railway)   약 3~5만원
DB (PostgreSQL)      약 1~2만원
전화번호 유지         약 1,500원
합계                 약 4~7만원
```

### 손익분기점 시뮬레이션 (어르신 100명, 1인 하루 10분)
```
변동비: 63원 × 10분 × 100명 × 30일 = 189,000원/월
고정비: 약 50,000원/월
총 원가: 약 24만원/월 → 1인당 약 2,400원/월
구독료 월 5,000~10,000원 기준 → 수익 확보 가능
```

## 주의사항 / 개발 원칙

- 어르신 사용자 기준: 디지털 문맹 가정, UX 극단적 단순화
- 개인정보: 음성 데이터 로컬 처리 우선, 클라우드 전송 최소화
- 사투리 응답 시 과장되지 않게 — 자연스러운 말투 수준 유지
- AI Hub 데이터 라이선스 확인 필수 (상업적 이용 조건)
- Twilio 한국 번호 등록 시간 여유 두고 진행 (수일 소요 가능)
